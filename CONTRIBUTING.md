# Contributing

Thanks for helping improve **md-sitemap**. This project is a voluntary open protocol, not a standards body.

## Ways to contribute

1. **Spec clarity** — Fix ambiguity in [SPEC.md](./SPEC.md) (prefer MUST/SHOULD/MAY language).
2. **Agent guidance** — Improve [AGENTS.md](./AGENTS.md) discovery, budgets, or degradation.
3. **Implementations** — Recipes in [IMPLEMENTATION.md](./IMPLEMENTATION.md) for more stacks.
4. **Adopters** — Add your production site to [sites.md](./sites.md) with a working `/md-sitemap.txt`.
5. **Examples** — Keep [examples/](./examples/) minimal and correct.

## Process

1. Open an Issue describing the problem or proposal.
2. For normative SPEC changes, explain impact on **Core** / **Full** conformance.
3. Prefer small PRs with complete sentences in the description.
4. Do not add product-specific data dumps from adopter sites into this repo.

## Style

- Oxford commas in prose.
- **Bold** for emphasis; backticks for paths, headers, and field names.
- Tables: use leading and trailing pipes; keep cells short.
- Copyright year tracks the project calendar year (currently **2026**).
